oak_abi

[−][src]Module oak_abi::label

Labels are defined at the ABI level because they need to be available:

  • to applications (e.g. via the SDK), so that they can be manipulated by running Oak Nodes
  • to the Runtime, so that they can be checked and enforced when data is exchanged between Nodes over Channels

In order to support this, Labels are serialized and deserialized across the ABI boundary, and can be read and write using ABI functions.
